Purpose and Introduction


The purpose of this report is to explore the workings of the File Transfer Protocol (FTP). Various sources will be utilized for research, which will be used to explain multiple facets of the FTP. Aspects of the FTP that will be explored include the message type, message sequence, message syntax and type of connection. Both written and visual representations will be made to assist the reader with understanding the material.

The FTP is a commonly used internet protocol used for the transfer of files between computers using Transmission Control Protocol/Internet Protocol (TCP/IP) connections. FTP was first introduced by Abhay Bhushan in RFC 114 published in April 1971 (Bhushan, 1971). This original version ran on the network control program ARPA network before the modern internet came into existence (Bhushan, 1971). The FTP went through several additional updates over the decades with the Transmission Control Protocol/Internet Protocol (TCP/IP) version being published in 1980 as RFC 765 (Postel, 1980). The most current iteration was published in October 1985 in RFC 959 (Postel and Reynolds, 1985).

There are several standards which complement the official RFC 959, including RFC 1579, RFC 2228 and RFC 2428 (Bellovin, 1994) (Horowitz, 1997) (Allman, 1998). FTP is bidirectional in that it allows both the uploading and downloading of files (Goerzen, 2004). As J. Goerzen argues in Foundations on Python Network Programming, “FTP is one of the oldest and most widely used protocols on the internet” (2004, p. 275).

There are many applications using the FTP protocol, including FTP web applications for file downloads, FTP clients for storage and file sharing, single file sharing or downloads, sharing non-sensitive information, FTP automation and business to business file transfer (BasuMallick, 2022).
